# Project: Objects, interfaces and classes

This project is a collection of TypeScript exercises that demonstrate various programming concepts related with generic interfaces and SOLID principles. Includes documentation generated using TypeDoc.

## Project Structure

The project is organized as follows:
```
├── dist/                  # Compiled JavaScript files and type definitions
├── docs/                  # Documentation generated by TypeDoc
│   ├── assets/            # Static assets for documentation (CSS, JS, icons)
│   ├── functions/         # Documentation for individual functions
│   ├── modules/           # Documentation for each module (exercise)
│   └── types/             # Documentation for custom types
├── src/                   # Source TypeScript files for each exercise
├── eslint.config.mjs      # ESLint configuration for linting
├── package.json           # Project dependencies and scripts
├── package-lock.json      # Lock file for dependencies
├── tsconfig.json          # TypeScript configuration
└── typedoc.json           # TypeDoc configuration for documentation
```

## Documentation

The project includes auto-generated documentation using **TypeDoc**. The documentation is located in the `docs/` directory and can be viewed by opening `docs/index.html` in a browser.

### How to Generate Documentation
1. Install TypeDoc (if not already installed):
   ```bash
   npm install typedoc --save-dev
   ```
2. Run the following command to generate documentation:
   ```bash
   npm run docs
   ```
3. Open docs/index.html to view the documentation.
   ```bash
   npm run documentation
   ```

## Getting Started
### Installation
1. Clone the repository:
   ```bash
   git clone git@github.com:ULL-ESIT-INF-DSI-2425/prct06-generics-solid-alu0101433943.git
   ```
2. Navigate to the project directory:
   ```bash
   cd prct06-generics-solid-alu0101433943
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
### Running the Exercises
Compile the TypeScript files:
   ```bash
   npm run build
   ```

### Linting, Formatting and Testing
The project uses ESLint, Prettier and Vitest for code quality, formatting and testing. Run the following commands:
- Lint the code:
   ```bash
   npm run eslint
   ```
- Format the code:
   ```bash
   npm run prettier
   ```
- Test the code:
   ```bash
   npm run test
   ```
- Test one particular exercise:
   ```bash
   npm run test ejercicio-X
   ```
